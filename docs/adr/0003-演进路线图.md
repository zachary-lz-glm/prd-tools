## Context（为什么做）

领导提出 4 个后续演进方向，要求结合全网企业级 AI 工程化优秀案例做可行性分析。

## Decision（4 个方向的判断）

| # | 方向 | 可行性 | 投入 | 收益 | 核心行业映射 |
|---|------|--------|------|------|-------------|
| 1 | 多轮构建优化前后端 reference | 高 | 中 | 高 | Anthropic Evaluator-Optimizer |
| 2 | 沉淀 B 端营销知识库 | 中高 | 高 | 长期战略 | Salesforce Agentforce |
| 3 | 优化能力面适配器 | 最高 | 低 | 中高 | Anthropic Routing |
| 4 | 三方视角 B 端整体 report + plan | 高 | 中 | 最高 | Anthropic Orchestrator-Workers |

## 路线图

```text
Phase 1（1-2 周）：方向 3 — 能力面适配器优化
  ├─ 低风险快速见效
  ├─ 动态适配器 + 项目级自定义能力面
  └─ 为后续方向打基础

Phase 2（2-4 周）：方向 1 — 多轮构建
  ├─ build-reference 支持 2-3 轮迭代
  ├─ 各层独立 reference + 跨层契约引用
  └─ Evaluator-Optimizer 模式实现

Phase 3（4-8 周）：方向 4 — 三方视角 Skill
  ├─ 依赖 Phase 2 的独立 reference
  ├─ Orchestrator-Workers 架构
  ├─ prd-distill-full skill 开发
  └─ 三方 report + plan 输出

Phase 4（持续）：方向 2 — 营销知识库
  ├─ 从 Phase 2 的多项目 reference 中提炼
  ├─ 随每次 prd-distill 持续回流
  └─ 长期战略资产
```

### 依赖关系

```text
方向 3（适配器优化）
    ↓ 不阻塞但增强
方向 1（多轮构建）────→ 方向 4（三方视角 Skill）
                              ↑ 依赖
方向 2（营销知识库）←──── 持续沉淀 ←──── 全程
```

## Consequences（影响）

### 收益

- 4 种 Anthropic Agentic Pattern 中有 3 种直接映射到演进方向，技术选型有据可依
- 分阶段递进：适配器精准 → reference 质量高 → 三方编排效果好

### 风险

- 缺少量化验收标准和 pilot 闭环（GPT-5.5 review 指出）
- 方向 2 初期投入大，需从多个项目的 reference 中提炼共性

## 各方向详情

### 方向 1：多轮构建

Anthropic Evaluator-Optimizer Pattern：Agent 生成初稿 → 评估器打分 → 迭代优化。

多轮意味着 token 成本约 2-3x，需设计收敛条件（boundary 违规数 < 阈值 或 最大轮次）。

### 方向 2：营销知识库

Salesforce Agentforce 的核心壁垒不是模型能力，而是垂直行业知识库的深度。

建议从 05-domain.yaml 的现有雏形出发，逐步抽取为独立的 marketing-knowledge-base/。

### 方向 3：能力面适配器

当前 layer-adapters.md 是静态文本，所有项目共用一套。

优化为：动态适配器（按技术栈选择）+ 项目级自定义 + 版本化 + 知识库反哺。

### 方向 4：三方视角 Skill

Orchestrator-Workers 架构：一个主控 skill 拆分任务 → 前端/BFF/后端子 Agent → 汇总三方 report + plan。

输出包含：PRD 理解、能力点映射、未映射 TODO、Test Case、Question、技术方案详情。
